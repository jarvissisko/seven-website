Overview

Built for rapid IoT prototyping

Seven is designed for engineers who need a cellular-connected development platform that doesn't slow them down — from first power-on to deployed prototype in hours, not weeks.

LTE Cellular, out of the box

The nRF9151 integrates a full LTE-M and NB-IoT modem with GNSS. No external cellular module, no ribbon cables — just one chip, one antenna, and your application.

mikroBUS™ expansion

A standard mikroBUS socket gives you access to 1,000+ Click Boards from MIKROE — sensors, displays, relays, motor drivers — without writing a single driver from scratch.

Wide-input power

6–36 V DC input means Seven works directly from industrial power rails, vehicle batteries, or solar panels — no custom power supply required for your first prototype.

Zephyr RTOS

Nordic's official SDK is built on the Zephyr Project. Real-time kernel, device tree, subsystems for networking, storage, BLE, and a proper build system — from day one.

Fully open hardware

Every schematic, PCB layout, and BOM is published under CERN-OHL-P v2. Fork it, manufacture it, improve it. No secret sauce. No license restrictions on derivatives.

Cloud-ready from the start

Pre-validated integration with AWS IoT Core through nRF Cloud and the nRF9151 modem. TLS 1.2 mutual auth, MQTT, and FOTA — the plumbing is handled.

Software

A mature stack, from day one

Seven runs on the Nordic nRF Connect SDK — a production-grade, well-documented stack built on top of Zephyr RTOS. Cellular, cloud, OTA — all handled.

Zephyr RTOS
nRF Connect SDK v2.x

Nordic's nRF Connect SDK is built on the Zephyr Project — a Linux Foundation open-source RTOS with a rich set of networking, storage, and peripheral drivers. Configured via device tree.

Zephyr 3.x CMake west DTS
LTE Modem
Integrated AT command interface

The nRF9151 modem exposes a standard AT command interface via UART, fully abstracted by the nRF Connect SDK modem library. LTE-M and NB-IoT band configuration, PSM, eDRX — all handled.

LTE-M NB-IoT PSM eDRX
AWS IoT Core
Pre-validated cloud integration

Publish telemetry, subscribe to commands, and manage device state through AWS IoT Core. Pre-validated with the nRF9151 using MQTT over TLS 1.2 with X.509 mutual authentication.

MQTT TLS 1.2 X.509 AWS IoT
OTA Firmware Updates
FOTA over LTE

Firmware-over-the-air (FOTA) updates are built into the nRF Connect SDK. Push new firmware from AWS IoT Jobs or nRF Cloud — devices update securely without physical access.

FOTA MCUboot nRF Cloud
GNSS & Location
Integrated GPS

The nRF9151 includes an integrated GNSS receiver. Acquire a GPS fix without additional hardware. Use A-GPS with nRF Cloud for faster time-to-first-fix, even in challenging conditions.

GPS A-GPS TTFF
Security
ARM TrustZone & secure storage

The Cortex-M33 includes ARM TrustZone technology. nRF9151 adds a hardware-backed key storage and a modem-side TLS stack — credentials never leave the secure enclave in plaintext.

Get Started

From unboxed to LTE in three steps

The quickest path to your first LTE-connected prototype using the nRF Connect SDK and Zephyr.

01

Set up the SDK

Install the nRF Connect SDK using the nRF Connect for Desktop toolchain manager. Installs Zephyr, west, and all required toolchains.

west init -m https://github.com/nrfconnect/sdk-nrf west update
02

Clone board files

The Seven board definition and DTS files live in the hardware repository. Clone it and point your west workspace at the out-of-tree board definition.

git clone https://github.com/id8-engineering/seven-hardware # Add to west.yml or use BOARD_ROOT
03

Build & flash

Build the included asset_tracker sample or start from a minimal hello_nrf9151 application. Flash via USB-C with the nRF Connect Programmer.

west build -b seven_nrf9151 -- -DEXTRA_CONF_FILE=overlay-lte.conf west flash
ℹ Documentation and sample applications are being prepared alongside the hardware release. Watch the GitHub repository for updates. Early adopters are welcome to open issues and contribute.
